# Pre-Test Checklist

Use this checklist before every physical test. A test is not approved unless every applicable line is complete. The local operator has final authority to stop the test.

## Test Identification

| Item | Status |
|---|---|
| Test ID created before setup. |  |
| Phase and objective documented. |  |
| One-variable change identified. |  |
| Metadata JSON file started. |  |
| Expected result written in one sentence. |  |

## Safety and Scope

| Item | Status |
|---|---|
| Test method is non-pyrotechnic and within repository scope. |  |
| Test area is clear of people, pets, and loose objects. |  |
| Fragment shield, backstop, or containment is in place if brittle materials are used. |  |
| PPE is available and worn as appropriate. |  |
| Emergency stop or lockout has been checked. |  |
| Local operator confirms authority to stop test at any time. |  |

## Equipment Readiness

| Item | Status |
|---|---|
| Sensor logger powers on. |  |
| CSV data file is being created. |  |
| Metadata file contains firmware and structure IDs. |  |
| Wide camera view covers entire test area. |  |
| Close camera view covers primary failure zone. |  |
| Lighting is sufficient. |  |
| Fixture is stable before test. |  |

## Go/No-Go Decision

| Decision | Notes |
|---|---|
| Go |  |
| No-go |  |
| Repeat dry-run first |  |

## Post-Test Capture

| Item | Status |
|---|---|
| Sensor data saved. |  |
| Metadata completed. |  |
| Camera filenames recorded. |  |
| Photos of final state captured. |  |
| Cleanup completed. |  |
| Result classified as pass, repeat, or fail. |  |
| GitHub issue updated. |  |
